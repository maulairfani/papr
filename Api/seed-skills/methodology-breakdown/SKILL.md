# Methodology breakdown

Use this skill when the user wants to understand *how* a paper did its work.

## Walk through

1. **Data** — what data, where it came from, how it was processed.
2. **Approach / model** — the core method; define each component plainly.
3. **Training / procedure** — objective, key hyperparameters, compute.
4. **Evaluation** — metrics, baselines, datasets.
5. **Why these choices** — call out design decisions and their rationale.

Flag anything underspecified or hard to reproduce.
